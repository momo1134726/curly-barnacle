# おじさんゲーム

## プロジェクト概要

様々な個性的なおじさんを収集・育成・コレクション共有・バトルさせるゲームアプリ。
Phaser 3 + TypeScript で開発。スマホ・タブレット・PC すべてに対応し、タッチ操作とキーボード操作の両方をサポートする。

### コンセプト
- 使いやすく、面白くて**くだらないがどこかかっこいい**トーン
- 流行を取り入れ、**定期的なおじさん追加**で継続的に盛り上げる
- UI・BGM・効果音にこだわったクオリティ

---

## ゲーム仕様

### おじさんの収集方法
- **ガチャ**（メイン収集手段）
- **ミッション達成**報酬
- **ログインボーナス**

おじさんは基本ガチャで入手し、ミッション・ログインボーナスは **ガチャチケット**（無料でガチャ1回を引ける通貨）を報酬として渡す方式。`src/systems/SaveSystem.ts` で所持数・チケット・通算ガチャ回数・ログイン履歴・ミッション達成状況を LocalStorage に永続化。

#### ミッション（`src/data/missions.ts`）
- ガチャ累計回数・コレクション種類数を条件に、達成でチケットを獲得
- `MissionScene` で進捗バーと受け取りボタンを表示

#### ログインボーナス（`src/data/loginBonus.ts`）
- 7日サイクルの連続ログイン日数に応じてチケット枚数が増加（`[1,1,2,1,2,3,5]`）
- `HomeScene` 表示時に1日1回判定し、初回ならダイアログで報酬を表示

### おじさんの構成要素

| 要素 | 内容 |
|------|------|
| 見た目 | 各おじさん固有（イラスト） |
| 固有スキル | 各おじさん固有（最大の個性） |
| 性格 | ステータス倍率に影響（複数種類） |
| 技 | プレイヤーが自由にカスタマイズ可能 |
| ステータス | スキルポイント（SP）を割り振って強化 |
| 相性 | おじさん間・技間の相性システム |

### ステータス・育成システム
- 決められた **スキルポイント（SP割り振り用）** を各ステータスに振り分けて強化
- 振り方のパターンによって **ボーナス強化**が発動（重複発動あり）
- **性格**によってステータス倍率が変化（1ステータス×1.1倍、対応する1ステータス×0.9倍）
- レアキャラは強力な固有スキルを持つが**強すぎない設計**
- どんなおじさんでもカスタム次第で強くなれるバランス

### ステータス一覧（8種）

| 記号 | 名称 | 内容 |
|------|------|------|
| HP | HP | 体力。0になると戦闘不能 |
| SP | SP | 技を使うために消費するポイント |
| A | 物理攻撃力 | 物理技のダメージに影響 |
| B | 物理防御力 | 物理技で受けるダメージを軽減 |
| SA | 特殊攻撃力 | 特殊技のダメージに影響 |
| SB | 特殊防御力 | 特殊技で受けるダメージを軽減 |
| SD | 素早さ | 行動順・回避に影響 |
| LUCK | 運 | クリティカル率・状態異常の成功率等に影響 |

> ※ ステータスの表示名（日本語ニックネーム）は後で追加予定

### 性格一覧（8種）

| 性格名（仮） | ×1.1 | ×0.9 |
|-------------|------|------|
| 根性系 | HP | SD |
| 技巧系 | SP | LUCK |
| 猛者系 | A | SB |
| 堅牢系 | B | SA |
| 念力系 | SA | B |
| 忍耐系 | SB | A |
| 俊足系 | SD | HP |
| 強運系 | LUCK | SP |

> ※ 性格名はおじさんらしい名前に後で変更予定

### SP割り振りボーナス一覧（10種・重複発動あり）

| ボーナス名 | 発動条件 | 効果 |
|-----------|---------|------|
| 猛攻 | A + SA に合計60%以上 | 技の威力 +12% |
| 物理特化 | A のみに50%以上 | 物理技威力 +20%、ただし B -10% |
| 念動力 | SA のみに50%以上 | 特殊技威力 +20%、ただし SB -10% |
| 鉄壁 | B + SB に合計60%以上 | 被ダメージ -15% |
| 根性 | HP に40%以上 | 最大HP +20%、HP50%以下で防御+10%追加 |
| 電光石火 | SD に35%以上 かつ A か SA に25%以上 | 先制率UP・クリティカル率 +10% |
| 技巧派 | SP に30%以上 | SP最大値 +25%・技コスト -10% |
| 幸運体質 | LUCK に35%以上 | クリティカル率 +15%・状態異常耐性 +20% |
| 万能型 | 全ステに最低10%ずつ | 全ステータス +7% |
| 忍耐 | HP・B・SB それぞれ20%以上 | 毎ターン最大HPの3%回復 |

### おじさんの種類
- 初期リリース：**30種類**（現在は名前・固有スキルともプレースホルダー、`src/data/ojisans.ts`）
- レアリティ：★3 / ★4 / ★5（内訳 15体 / 10体 / 5体）
- 定期的に新おじさんを追加

### ガチャ排出率（やさしめ設定）

| レア度 | 排出率 |
|--------|--------|
| ★5 | 8% |
| ★4 | 32% |
| ★3 | 60% |

`src/data/gachaRates.ts` で管理

### 相性システム（職業系タイプ・8種）

各タイプが1つに強く・1つに弱い循環型。相性倍率：有利×1.5 / 不利×0.67

| タイプ | 強い相手 | 弱い相手 |
|--------|---------|---------|
| 無職 | インテリ | 会社員 |
| 会社員 | 無職 | 職人 |
| 職人 | 会社員 | 芸能系 |
| 芸能系 | 職人 | 自営業 |
| 自営業 | 芸能系 | ヤンキー |
| ヤンキー | 自営業 | 体育会系 |
| 体育会系 | ヤンキー | インテリ |
| インテリ | 体育会系 | 無職 |

循環：無職 → インテリ → 体育会系 → ヤンキー → 自営業 → 芸能系 → 職人 → 会社員 → 無職

## マルチデバイス対応

### 画面サイズ・スケーリング

- `Phaser.Scale.FIT` + `autoCenter: Phaser.Scale.CENTER_BOTH` でアスペクト比を維持しつつ全画面フィット
- ベース解像度：**390×844**（iPhone縦持ち基準、スマホファースト）
- UI 要素は `scene.scale.width` / `scene.scale.height` を参照して配置する（ハードコード禁止）
- `resize` イベントを購読してレイアウトを再計算する

```typescript
scale: {
  mode: Phaser.Scale.FIT,
  autoCenter: Phaser.Scale.CENTER_BOTH,
  width: 390,
  height: 844,
}
```

### タッチ操作

- `scene.input.on('pointerdown' / 'pointermove' / 'pointerup')` で統一処理
- ボタン等のヒット領域は最低 **48×48px** 以上確保
- マルチタッチが必要な場合は `scene.input.addPointer(n)`

### キーボード操作

- タッチと排他にせず、どちらでも同じアクションが動くように実装

## UIトーン・デザイン指針

- **くだらないがかっこいい**：ふざけすぎず、でも笑えるバランス
- おじさんのビジュアルは個性全開、UIは見やすくスッキリ
- ガチャ演出・育成時のアニメーションは**しっかり作り込む**
- BGM：場面ごとに雰囲気を変える（ガチャは高揚感、育成はまったり等）
- 効果音：ボタン・ガチャ・レベルアップなど細かく設定

### 配色

鮮烈レッドベース × ゴールド × シアン/マゼンタの「パチンコ演出を参考にした目を引く配色」。`constants.ts` の `COLORS`（Phaser用16進数値）/ `HEX`（文字列）を参照。

| 用途 | 色 | 値 |
|------|-----|-----|
| 背景 | 鮮烈レッド | `#4a0a14` |
| 背景（濃） | ほぼ黒レッド | `#2e060d` |
| パネル・カード | レッドパネル | `#6b0f1f` |
| パネル枠線 | 明るいレッド | `#a31530` |
| メインCTA（ボタン） | ゴールド | `#ffb627` |
| 見出し・タイトル | マゼンタピンク | `#ff2e7a` |
| 差し色（無料/チケット系） | シアン | `#29d8ff` |
| 本文 | 白 | `#ffffff` |
| 補助テキスト | 淡いピンクグレー | `#e0a8b3` |

ゴールド／シアン等の明るい背景に乗せるテキストは可読性のため `HEX.BG_DARK` を使う。

### フォント・テキスト表示

- 見出し：`Mochiy Pop One`（インパクト重視のポップ体）／本文：`Zen Maru Gothic`（丸ゴシックで読みやすさ重視）。`constants.ts` の `FONT_HEADING` / `FONT_BODY` を参照
- Google Fonts を `index.html` で読み込み、`main.ts` で `document.fonts.load()` の完了を待ってから `new Phaser.Game()` を起動する（先に起動するとCanvas描画がフォールバックフォントのまま固定されるため）
- Phaser Text は日本語の濁点・半濁点・絵文字で上部が見切れやすいため、すべてのテキストスタイルに `padding: { top: ..., bottom: ... }` を付与する
- CSSで拡大表示される分テキストがぼやけるため、すべてのテキストスタイルに `resolution: TEXT_RESOLUTION`（`constants.ts`、devicePixelRatio基準）を付与してシャープに保つ

## コーディング規約

- シーンは `Phaser.Scene` を継承したクラスで実装し、`create()` / `update()` を分離する
- マジックナンバーは `constants.ts` に集約する
- アセットキーは `const enum` で管理して typo を防ぐ
- `update(time, delta)` では `delta`（ms）を使ってフレームレート非依存で計算する
- データ（おじさん・技・性格）はコードと分離して `src/data/` に集約する
- セーブデータは `SaveSystem` 経由のみでアクセスする（直接 LocalStorage 操作禁止）

---

## パフォーマンス指針

- テクスチャアトラスでドローコール削減
- おじさんカードは `Phaser.GameObjects.Group` でプール管理
- 画像は WebP 形式を推奨
- 物理演算は基本使用しない（コレクション・育成系のため）
